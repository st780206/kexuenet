chrome不能翻墙怎么安装proxy插件
在 Chrome 无法翻墙的情况下，你无法访问 Chrome 网上应用店，可以采用以下两种离线安装方案：
方法一：通过“开发者模式”安装离线包（最推荐）
这种方法不需要翻墙，只需从国内可访问的渠道下载扩展文件（.crx 或 .zip）。
下载离线文件：
在其他能上网的设备或通过国内镜像站（如 极简插件 或 扩展迷）下载 SwitchyOmega 的 .zip 或 .crx 包。
也可以直接访问 GitHub SwitchyOmega Releases 下载最新版。
开启开发者模式：
在 Chrome 地址栏输入 chrome://extensions/ 并回车。
打开右上角的 「开发者模式」 开关。
安装插件：
如果是 .zip 文件：先解压，点击 「加载已解压的扩展程序」，选择该文件夹。
如果是 .crx 文件：直接将文件拖拽到这个扩展页面。如果提示“无法从该网站添加”，请将 .crx 后缀名改为 .zip，解压后再通过“加载已解压”安装。
方法二：利用本地 PAC 脚本或系统代理（无需插件）
如果你暂时找不到插件包，可以直接通过 Chrome 的 内置设置 使用你的 OpenClash/PassWall 代理：
打开 Chrome [设置] -> [系统] -> [打开您计算机的代理设置]。
在 Windows/macOS 的系统代理设置中：
开启 「手动设置代理」。
地址：192.168.1.11（你的路由器或代理主机 IP）。
端口：8088（你之前测试通过的 SOCKS 端口）。
保存后，Chrome 会直接遵循系统代理，此时你就可以访问 Chrome 应用商店 搜索并安装 SwitchyOmega 了。
方法三：命令行强行启动
你可以通过命令行带参数启动 Chrome，强制它使用代理：
bash
# Windows 示例 (在运行或 CMD 中执行)
chrome.exe --proxy-server="socks5://192.168.1.11:8088"
请谨慎使用此类代码。

关键提示：安装好 SwitchyOmega 后，记得在插件里新建一个 Proxy 情景模式，协议选 SOCKS5，地址填 192.168.1.11，端口填 8088，最后点击 [应用选项] 才能生效。
